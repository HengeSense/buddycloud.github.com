buddycloud DNS

Estimated time: 10 mins

Configure

Log into your registrar or edit your DNS zone file (at Godaddy this is called the DNS Manager). In this example we use 1.2.3.4 as your server address.

Not using Bind for DNS? See Google's guide on how to setup a TXT record. 


; SRV records (below) must point to A records (never CNAMES)
my-server.EXAMPLE.COM.                     IN A            1.2.3.4

; this is the website virtual host                            
buddycloud.EXAMPLE.COM.                    IN CNAME        my-server.EXAMPLE.COM.           
; non-logged in web users   
anon.EXAMPLE.COM.                          IN CNAME        my-server.EXAMPLE.COM.

; enable clients to discover the API endpoint   
_buddycloud-api._tcp.EXAMPLE.COM.          IN TXT          "v=1.0" "host=buddycloud.EXAMPLE.COM" "protocol=https" "path=/api" "port=443"

; SRV records must point to A records
_xmpp-server._tcp.EXAMPLE.COM.             IN SRV 5 0 5269 my-server.EXAMPLE.COM.              
_xmpp-server._tcp.buddycloud.EXAMPLE.COM.  IN SRV 5 0 5269 my-server.EXAMPLE.COM. 
_xmpp-server._tcp.anon.EXAMPLE.COM.        IN SRV 5 0 5269 my-server.EXAMPLE.COM.

Test

Use the dig command to test your DNS 


#check that the buddycloud server can be resolved as an A record
dig -t A my-server.EXAMPLE.COM +short
1.2.3.4

#check the CNAME records
dig -t A buddycloud.EXAMPLE.COM +short
1.2.3.4
dig -t A anon.EXAMPLE.COM +short
1.2.3.4

#check the server SRV record
#dig -t srv _xmpp-server._tcp.EXAMPLE.COM +short
5 0 5269 my-server.EXAMPLE.COM.

#check the server SRV record
"v=1.0" "host=api.buddycloud.org" "protocol=https" "path=/" "port=443"

Firewall Setup

Estimated time: 2 mins

Double check your firewall rules. They should look something like this: 


# inbound webclient connections (bounced to SSL)
iptables -A INPUT  -m state --state NEW -s 0.0.0.0/0 -d <your server address> -p tcp --dport 80   -j ACCEPT 
# inbound webclient connections
iptables -A INPUT  -m state --state NEW -s 0.0.0.0/0 -d <your server address> -p tcp --dport 443  -j ACCEPT 
# inbound XMPP client connections
iptables -A INPUT  -m state --state NEW -s 0.0.0.0/0 -d <your server address> -p tcp --dport 5222 -j ACCEPT
# inbound connections from other buddycloud servers
iptables -A INPUT  -m state --state NEW -s 0.0.0.0/0 -d <your server address> -p tcp --dport 5269 -j ACCEPT
# outbound connections to other buddycloud servers
iptables -A OUTPUT -m state --state NEW -s <your server address> -d 0.0.0.0/0 -p tcp --dport 5269 -j ACCEPT

If your hosting provider has its own firewall in addition to the firewall on your server (like on Amazon EC2 instances), make sure to also open TCP ports 80, 43, 5222 and 5269 there.

Prosody Setup

Estimated time: 10 mins

  • alternative: Installing buddycloud on ejabberd
  • alternative: official Prosody install guide (good for older distributions)
  • alternative: Prosody with LDAP authentication
  • alternative: Installing buddycloud on openfire

Dependencies

  • Install the Prosody server and the OS Dependencies 


apt-get install prosody liblua5.1-sql-postgres-2 lua-dbi-postgresql postgresql

Configure Server

Edit /etc/prosody/prosody.cfg.lua, with the following settings. 


modules_enabled = {

 -- "roster";          -- Allow users to have a roster.
 "saslauth";           -- Authentication for clients and servers. Recommended if you want to log in.
 "tls";                -- Add support for secure TLS on c2s/s2s connections
 "dialback";           -- s2s dialback support
 "disco";              -- Service discovery
 "time";               -- Let others know the time here on this server
 "ping";               -- Replies to XMPP pings with pongs
 "posix";              -- Do posixy things
 "register";           -- Allow users to register on this server using a client and change passwords
 "watchregistrations"; -- Alert admins of registrations (check "admins" option below too)
 "compression";        -- Enable mod_compression
 "legacyauth";         -- Needed by strophe
 

storage                 = "internal"
admins                  = { "your-username@EXAMPLE.COM" }  -- who can admin this domain / who receives registration alerts
pidfile                 = "/var/run/prosody/prosody.pid"
log                     = {{ levels = { min = "error" }, to = "file", filename = "/var/log/prosody/prosody.err" } 
                          { levels = { min = "info"  }, to = "file", filename = "/var/log/prosody/prosody.log" };}

registration_whitelist      = { "127.0.0.1" }
whitelist_registration_only = true

VirtualHost "EXAMPLE.COM"
 authentication        = "internal_hashed"
 allow_registration    = true 
 anonymous_login       = false
 ssl                   = { key = "/etc/apache2/certs/EXAMPLE.COM.key"; certificate = "/etc/apache2/certs/EXAMPLE.COM.pem" }

-- for non-logged in browsing of open channels.
VirtualHost "anon.EXAMPLE.COM"
 authentication        = "anonymous"
 allow_registration    = false
 anonymous_login       = true
 disallow_s2s          = true

-- this is the connection used by the buddycloud server
Component "buddycloud.EXAMPLE.COM"
 component_secret      = "tellnoone"

restart the server 


/etc/init.d/prosody restart

Test

test the server is running 


# from a remote machine:
telnet EXAMPLE.COM 5269
# you should receive a connection. 

# from a remote machine:
telnet EXAMPLE.COM 5222
#you should receive a connection.

Debug


tail -F /var/log/prosody/prosody.err \
/var/log/prosody/prosody.log

buddycloud Server

Estimated time: 10 mins

  • Alternative: Running a buddycloud topics server
  • Alternative: build the buddycloud server yourself
  • Alternative: manually create the buddycloud server database

Install


sudo apt-get install software-properties-common python-software-properties; #needed for PPA access
sudo add-apt-repository ppa:chris-lea/node.js-legacy # to retrieve nodejs version 0.8.23
sudo apt-get update
sudo apt-get install dbconfig-common \
                    libpq5  libicu48  libicu-dev libssl1.0.0 \
                    subversion libexpat1 libexpat1-dev openssl \
                    build-essential libssl-dev curl unzip nodejs
node --version 
# The buddycloud-server requires nodejs 0.8.*

Download the latest buddycloud-server package from downloads.buddycloud.com 


sudo dpkg -i buddycloud-server*.deb;

Configure server


sudo vim /etc/buddycloud-server/config.js

Start Server


sudo /etc/init.d/buddycloud-server start

Test


sudo tail -F /var/log/buddycloud-server/buddycloud-server.log

Debug database


# Test the database is installed
psql -h 127.0.0.1 --username buddycloudserver -d buddycloudserver -c "select * from nodes;"
# Password for user buddycloudserver: (look at /etc/buddycloud-server/config.js to find out)
node

(0 rows)
# this means that your buddycloud server database schema been installed successfully.

buddycloud API Server

Estimated time: 6 mins

SSL and API requests

Chrome and Firefox will not work with self signed certificates when used for API calls. Ordinarily you would receive a certificate warning. But for API requests, this does not happen and requests are silently drop. You have two options: # run without SSL on your API server

  • get a signed certificate

    • Dependencies

    
apt-get install apache2 apache2.2-common
# ensure you have the correct modules enabled in Apache
# a2enmod rewrite proxy_http ssl headers expires deflate
sudo service apache2 reload

    Install

    
sudo apt-get install software-properties-common python-software-properties; #needed for PPA access
sudo add-apt-repository ppa:chris-lea/node.js-legacy # to retrieve a more current nodejs version
sudo apt-get update
sudo apt-get install libpq5  libicu48  libicu-dev libxml2-dev libssl1.0.0 \
                    subversion libexpat1 libexpat1-dev openssl \
                    build-essential libssl-dev curl unzip nodejs
node --version # The buddycloud API requires nodejs 0.8.*

    Download the latest buddycloud-http-api package from downloads.buddycloud.com 

    
sudo dpkg -i buddycloud-http-api*.deb;

    Configure Server

    
sudo vim /etc/buddycloud-http-api/config.js

    Start Server

    
sudo /etc/init.d/buddycloud-http-api start

    Test

    • Open http://buddycloud.EXAMPLE.COM/api/team@topics.buddycloud.org/content/posts in your browser.
      • You should see an XML feed of posts

    Debug

    
tail -F /var/log/buddycloud-http-api/buddycloud-http-api.log

    buddycloud Webclient

    Estimated time: 5 mins

    • Alternative: buddycloud Apache virtual host setup
    • Alternative: buddycloud nginx setup
    • Alternative: buddycloud lighttpd setup
    • Alternative: buddycloud SSL setup

    Install

    Download the latest buddycloud-webclient package from downloads.buddycloud.com 

    
sudo dpkg -i buddycloud-webclient*.deb;

    Configure

    
sudo vim /etc/apache2/sites-available/buddycloud-apache-virtual-host
sudo vim /usr/share/buddycloud-webclient/config.js
a2ensite buddycloud-apache-virtual-host
sudo service apache2 reload

    Test

    • check your SSL setup
    • open https://EXAMPLE.COM/ in your browser: you should see the login screen
    • open https://EXAMPLE.COM/team@topics.buddycloud.org in your browser: you should see an open channel displayed

    Debug

    
tail -F /var/log/apache2/EXAMPLE.COM-error.log \ 
       /var/log/apache2/EXAMPLE.COM-error.log \
       /var/log/apache2/api.EXAMPLE.COM-error.log \
       /var/log/apache2/api.EXAMPLE.COM-error.log

    Final steps

    You are done!

    Log-into your buddycloud node at https://EXAMPLE.COM/ and follow your first channels. And don't forget to add your server to the list of buddycloud enabled domains.
